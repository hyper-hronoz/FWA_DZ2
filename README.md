# Домашняя работа №2

Сравнение скорости поиска кратчайшего пути в графе дорог в браузере:

- `JavaScript`
- `WebAssembly`, собранный из `C++`

В обеих реализациях используется алгоритм Дейкстры для взвешенного связного графа с неотрицательными весами.

## Что делает приложение

1. Генерирует большой случайный связный граф дорог.
2. Выбирает случайные вершины старта и финиша.
3. Один раз строит `CSR`-представление графа.
4. Несколько раз запускает решение на `JS` и на `WASM`, замеряя время через `performance.now()`.
5. Сравнивает среднее время выполнения и проверяет совпадение результатов.

## Запуск

Требования:

- `Node.js`
- `clang++` с поддержкой target `wasm32`
- `wasm-ld`/`lld`

Команды:

```bash
npm install
npm run dev
```

При старте `dev` и `build` автоматически запускается сборка `WASM`.

## Сборка production

```bash
npm run build
npm run preview
```

## Параметры эксперимента

- `Количество вершин`: от `10 000`
- `Средняя степень`: плотность графа
- `Повторов замера`: сколько раз повторяется тест
- `Seed генератора`: чтобы воспроизводить один и тот же граф

## Методика сравнения

- На каждом запуске создаётся один общий набор входных данных.
- Для всех повторов используется один и тот же граф и одна и та же пара вершин.
- `CSR` готовится один раз, а не перестраивается перед каждым замером.
- Перед серией измерений выполняется прогрев обеих реализаций.
- Порядок запусков чередуется между `JS` и `WASM`, чтобы уменьшить влияние прогрева и шума.
- После каждого прогона проверяется совпадение найденного расстояния, длины пути и самого маршрута.
