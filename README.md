# Домашняя работа №2

Сравнение скорости поиска кратчайшего пути в графе дорог в браузере:

- `JavaScript`
- `WebAssembly`, собранный из `Rust`

В обеих реализациях используется алгоритм Дейкстры для взвешенного связного графа с неотрицательными весами.

## Что делает приложение

1. Генерирует большой случайный связный граф дорог.
2. Выбирает случайные вершины старта и финиша.
3. Запускает решение на `JS` и на `WASM`.
4. Несколько раз замеряет время через `performance.now()`.
5. Сравнивает среднее время выполнения и проверяет, что оба решения дают один и тот же результат.
6. Строит визуализацию подграфа, где выборка вершин и layout рассчитываются в `WASM`.

## Запуск

Требования:

- `Node.js`
- `Rust`
- доступ к `cargo`

Команды:

```bash
npm install
npm run dev
```

При старте `dev` и `build` автоматически запускается сборка `WASM`.

## Сборка production

```bash
npm run build
npm run preview
```

## Параметры эксперимента

- `Количество вершин`: от `10 000`
- `Средняя степень`: плотность графа
- `Повторов замера`: сколько раз повторяется тест
- `Seed генератора`: чтобы воспроизводить один и тот же граф

## Методика сравнения

- На каждом запуске создаётся один общий набор входных данных.
- `JS` и `WASM` получают один и тот же граф и одну и ту же пару вершин.
- Порядок замеров чередуется, чтобы снизить влияние прогрева.
- После каждого прогона проверяется совпадение найденного расстояния и длины маршрута.

Важно: в измерение `WASM` включён и вызов функции из JavaScript, и передача массивов в `WASM`. Это ближе к реальному сценарию использования в браузере.

## Визуализация графа

- Полный граф на десятках тысяч вершин плохо читается на экране, поэтому визуализируется подграф.
- `WASM` выбирает репрезентативную выборку вокруг стартовой и целевой вершины.
- `WASM` вычисляет координаты вершин force-directed layout.
- Браузер рисует уже готовые координаты на `canvas`.
